# CivStudio Web

> ‚öÝÔ∏è **Proof of concept.** This map/visualization is a prototype demo of the engine, not a
> finished product ‚Äî expect rough edges and breaking changes.

Read-only presentation layer for the headless engine. Nothing here runs during a
simulation; each page is a **consumer** of a run's output under `output/<seed>/`,
so it can never perturb the seed-reproducible run.

It is laid out as an ordinary static site ‚Äî `index.html` + `styles.css` + `app.js`
(a thin entry over the ES modules in `js/`, documented under **Files** below), with the
run's data in a generated `data.js` and the terrain as a real image asset under
`assets/`. `build.mjs` writes both. No bundler or dependencies ‚Äî just Node for the build.

## The WorldMap (with a live Caravans view)

The page is a **WorldMap** of the whole imported world (all ~4,710 land provinces),
rendered from the engine's **real terrain** ‚Äî the recolored `terrain.bmp` ‚Äî with all
province polygons, hover-to-read, pan/zoom (the map **wraps east-west** ‚Äî it is a
cylinder), province search, a full province-detail sidebar, and the per-plot terrain zoom
(textures, hillshade, rivers, features) on any province. Boundaries are **zoom-banded**:
zooming out fades province borders into region ‚Üí super-region ‚Üí continent outlines (dissolved
polygons precomputed by `TierBorderExporter`), each with its label tier. A **World | Political |
Caravans** toggle switches modes. **Political mode** colours each land province by its owning
Anbennar nation at the game-start bookmark (with hover/sidebar showing nation ¬∑ culture ¬∑ faith),
zoom-banded so the fill fades to the physical terrain as you dive in ‚Äî see
[`docs/political-map.md`](../docs/political-map.md). **Caravans** (`js/overlays/live.mjs`) is the
**live** view: it subscribes to the running spectator **server** over SSE and draws the current
session on the terrain ‚Äî the colony and its marching caravans in real time ‚Äî with a HUD (session
state, colony stats, the **taxation** command) and the top-bar clock/play/speed driving the hosted
session; see [`docs/client-server.md`](../docs/client-server.md). (There is no recorded-run replay:
the caravans come live from the server, so `build.mjs` and `data.js` are run-independent.)

The whole world's per-plot data is generated by `WorldPlotGenerator` (a dev tool,
`mvn -q compile exec:exec -Dsim.main=com.civstudio.geo.export.WorldPlotGenerator`) into
the gitignored `map/provinces/*.json.gz`; `build.mjs` concatenates those already-gzipped
grids into a single `assets/plots.pack` (with a byte-offset `plotIndex` in `data.js`),
range-fetched per province at zoom (see below).

The map leads with the **real EU4 terrain** ‚Äî the recolored `terrain.bmp` the engine imports ‚Äî
with **province polygons**, province labels, and the zoom-banded tier boundaries. It is
**draggable to pan and scrollable to zoom** (plus on-screen zoom controls); hovering picks the
province by its real outline (point-in-polygon). The **Caravans** view then overlays the live
session (colony marker + marching caravans with trails) drawn from the server feed.

**Terrain zoom (a base WorldMap layer).** Past ~5√ó zoom the blurred continent raster
resolves into the **real Civ4 terrain, plot by plot** ‚Äî each province's canonical plot
grid (`src/main/resources/map/provinces/<id>.json.gz`) is packed by the build into one
`assets/plots.pack` and **range-fetched on demand** (its byte range comes from
`BUNDLE.plotIndex`, and the fetched slice ‚Äî a standalone gzip member ‚Äî is gunzipped
in-page via `DecompressionStream`), rasterised to a 1px/plot offscreen canvas
(terrain colour from the Civ4 textures, relief shading, rivers, feature tint) and
blitted crisp under the map overlays. Past ~16√ó the flat tiles give way to **real
ground textures** ‚Äî each terrain's Civ4 *detail* texture, recoloured to its terrain
colour and baked into `assets/terrain-tiles-<seed>.png`, drawn per plot as a continuous
repeating pattern, with higher-`LayerOrder` terrains **feathered over their lower
neighbours** at shared edges (the Civ4 16-way blend, adapted to the raster). Relief is
a real **hillshade** from the imported `heightmap.bmp` ‚Äî each plot carries its elevation
(`ProvincePlotStore` ‚Üí `Plot.elevation`), and the renderer shades slope normals against a
NW sun and snow-caps the highest ground. **Rivers** draw as a water-textured ribbon
(baked from the Civ4 river art into `assets/river-<seed>.png`) that follows the plot
network and **tapers by the river's authored width** ‚Äî recovered from `rivers.bmp` into
each plot's river code rather than flattened to a flag (see `docs/river-rendering.md`).
Zoom reaches 256√ó. This layer is
mode-agnostic ‚Äî every overlay (Political, the live Caravans view) draws on top of it. Deep
link to a province at a zoom with `index.html#p=<id>&z=<zoom>`.

## Run it

Open `web/index.html` in a browser ‚Äî it loads `styles.css` + `data.js` + `app.js`
and the `assets/terrain-<seed>.png` image alongside it. Works straight off disk for the
core WorldMap + caravan replay; the **per-plot terrain zoom needs an HTTP server** (it
range-fetches `plots.pack`, which `file://` blocks) ‚Äî see the `file://` caveat below.

### Regenerate

The build distils the committed map resources ‚Äî the province map
(`src/main/resources/map/provinces.json`), the outlines (`borders.json`, `tierborders.json`),
the geographic hierarchy, and the tech tree ‚Äî into one JSON bundle written to `data.js`, and
bakes a dark-tinted crop of the real terrain raster (`data/anbennar/terrain.bmp`) into a real
image at `assets/terrain-<seed>.png` that the page references. It is **run-independent**: the
live caravans come from the server, so no `output/<seed>` run is needed (the `seed` arg only
names the baked terrain asset).

`borders.json` (the per-province polygon outlines) and `tierborders.json` (the dissolved
continent / super-region / region outlines) are committed map resources, so they are normally
already present. Rebuild them only if the map sources change:

```bash
mvn -q compile exec:exec -Dsim.main=com.civstudio.geo.export.ProvinceBorderExporter
mvn -q compile exec:exec -Dsim.main=com.civstudio.geo.export.TierBorderExporter
```

`build.mjs` copies `tierborders.json` through to the committed `assets/tiers.json` the page
lazy-fetches; the tier outlines draw **zoom-banded** (region ‚Üí super-region ‚Üí continent) with
province borders fading out under them as you zoom out (`web/js/overlays/tiers.mjs`).

```bash
node web/build.mjs            # -> web/data.js + web/assets/terrain-24601.png (no run needed)
```

**Everything the
site serves is committed** (non-LFS): `data.js` and the whole `assets/` folder ‚Äî the terrain
crop, the texture atlas, `plots.pack`, and every baked tile (`sea/shore/foam/ice/river/
bonus-icons-<seed>.png`), ~20 MB in all. So a fresh clone is immediately viewable and the
terrain zoom works without rebuilding. `node web/build.mjs` regenerates all of them in place
(from the plot grids + the Civ4 art) ‚Äî run it and commit the result whenever the engine, the
source rasters, or the build change. Only the heavy build *inputs* stay out of git: the
per-province plot grids `map/provinces/*.json.gz` (hundreds of MB; `WorldPlotGenerator` builds
and caches them for the build).

**`file://` caveat.** The terrain zoom now range-fetches `plots.pack`, and browsers
block `fetch()` on `file://`, so opening `index.html` straight off disk shows the core
WorldMap + caravan replay but **not** the per-plot terrain zoom (provinces stay the
blurred raster). Serve the folder over HTTP ‚Äî `npx serve web`, `swa start ./web`, or the
deployed site ‚Äî to get the zoom. This is the deliberate cost of packing the old loose
`assets/plots/<id>.js` `<script>` files (which did work off disk) into one range-fetched
file.

### Files

| file | role |
|------|------|
| `index.html` | the page markup ‚Äî links `styles.css`, `data.js`, `app.js`, and the terrain image |
| `styles.css` | all styling (theme tokens, layout, map chrome, sidebar) |
| `app.js` | **entry** ‚Äî a `type="module"` script that loads the `js/` modules and boots |
| `js/*.mjs` | the app, split into ES modules (see below) |
| `data.js` | **generated** ‚Äî `window.BUNDLE`, the map/geo/tech data (run-independent; no imagery inlined) |
| `assets/terrain-<seed>.png` | **generated** ‚Äî the dark-tinted terrain crop, a real image asset |
| `build.mjs` | reads the committed province map + outlines (`borders.json`/`tierborders.json`) + geographic hierarchy, writes `data.js`, `assets/tiers.json`, and the terrain PNG (run-independent) |
| `js/techtree.mjs` | the **technology tree** ‚Äî a full-screen modal over the map (map `paint()` pauses while it is up). Cards laid out on the C2C `iGridX`/`iGridY` grid, hairline SVG prereq elbows (solid AND / dashed OR), era-tab jumps, zoom + custom scrollbars, and a hover-ancestry gold highlight. Loads `assets/techs.pack` on first open |
| `js/shortcuts.mjs` | the single declarative source for the **bottom shortcut bar** (`#hotbar`) ‚Äî renders the hint chips (F7 tech tree, Space, F, 0, WASD, Esc). Key handlers stay with their feature (map nav in `panel.mjs`, F7 in `techtree.mjs`); this owns only the display |
| `gamefont.mjs` | shared reader for `GameFont_120.tga` glyph geometry (icon grid @ y=72, 25px cells) ‚Äî used by the bonus-icon bake (`build.mjs`) and the research beaker + future commerce/yield symbols (`build-techs.mjs`) |
| `build-techs.mjs` | **generated ‚Üí `assets/techs.pack` + `tech-icons.webp` + `tech-beaker.webp`** ‚Äî gzips the engine's `src/main/resources/techs.json` (tech graph + English names, enriched with a per-tech icon rect), bakes the tech-button icon sprite sheet from the vendored Civ4 art, and extracts the research-beaker glyph; run-independent (needs no `output/<seed>` run) |
| `assets/techs.pack` | **generated** ‚Äî the whole tech graph (grid coords, eras, advisors, prereqs, name/help/quote, `icon` rect), fetched and gunzipped in-page via `DecompressionStream` |
| `assets/tech-icons.webp` | **generated** ‚Äî one sprite sheet (16-wide, 64px cells) of the real Civ4 tech-button icons; a tech's `icon:[x,y,w,h]` indexes into it (the ~47 techs whose icon is a vanilla-BTS file C2C doesn't ship fall back to an advisor-colour chip) |
| `assets/tech-beaker.webp` | **generated** ‚Äî the GameFont research-beaker (the cost unit; blue = science, the human tree's default) |

**Modules** (`web/js/`). `app.js` was one ~1,100-line classic script; it is now a thin
entry that imports five ES modules in dependency order. Module scripts are deferred, so
the classic `data.js` still sets `window.BUNDLE` first. The site stays dependency-free ‚Äî
ES modules need no bundler, but they do need HTTP (the `file://` caveat above already
applied because of `plots.pack`). Load order: `core ‚Üí plots ‚Üí labels ‚Üí main ‚Üí panel`.

| module | role |
|--------|------|
| `core.mjs` | data prep, the lon/lat‚Üípixel‚Üíscreen **projection** + pan/zoom **camera**, the shared mutable **state object `S`**, constants (`K_PLOT`/`K_TEX`/‚Ä¶), and small utils. Everyone imports it. |
| `plots.mjs` | the per-plot **terrain-zoom layer** (range-fetch, rasterise, flat tiles ‚Üí textures ‚Üí hillshade ‚Üí **river ribbon** ‚Üí features) and the elevation **movement-cost overlay**. |
| `labels.mjs` | map text: province names and the **zoom-banded geographic tiers** (continent ‚Üí super-region ‚Üí region). |
| `main.mjs` | the **`draw()` orchestrator** (including the cylindrical **wrap** ‚Äî it renders the scene once per on-screen world copy), the camera raster, zoom, and deep-link. |
| `panel.mjs` | all **DOM interaction**: the sidebar (province detail, world summary), search, button tooltips, the live clock/play/speed (which drive the hosted session), and pointer/keyboard events. |

The eight primitives read *and* written across module boundaries (`mode`, `selected`,
`selectedProv`, `hoverProv`, `curT`, `showHeat`, `showCost`, `dragging`) plus the render
versioning live on the shared `S` object in `core`, since ES-module bindings can be read
across modules but not reassigned from outside their defining module.

The province outlines come from `ProvinceBorderExporter` (in the engine), which traces
each land province's silhouette from `provinces.bmp` into a simplified polygon and
writes `src/main/resources/map/borders.json` ‚Äî canonical geometry any UI can reuse.

## Deploy (live at https://anbennar.civstudio.com)

The site is hosted on **Azure Static Web Apps** (Free tier) in the **CivStudio**
subscription ‚Äî resource group `rg-anbennar-web`, app `anbennar-worldmap`.

**Deploys are automatic** via the GitHub Action
[`.github/workflows/deploy-web.yml`](../.github/workflows/deploy-web.yml): on every push
to `master` touching `web/` (and via a manual **Run workflow** button) it simply **uploads the
committed `web/` folder as-is** with [`Azure/static-web-apps-deploy`](https://github.com/Azure/static-web-apps-deploy)
‚Äî no Java engine, no `build.mjs`, no LFS, since `data.js` and every baked asset (including the
~17 MB `plots.pack`) are checked in. So a deploy ships exactly what is committed; **regenerate
the assets locally (`node web/build.mjs`) and commit them** whenever the engine, source rasters,
or build change. The deploy token lives in the `AZURE_STATIC_WEB_APPS_API_TOKEN` repo secret (set
once from `az staticwebapp secrets list -n anbennar-worldmap -g rg-anbennar-web`).

To **deploy by hand** (a local build machine, e.g. to test an unpushed change) the same
folder ships straight with the [SWA CLI](https://github.com/Azure/static-web-apps-cli):

```bash
node web/build.mjs                                    # produce data.js + assets/plots.pack (+ terrain PNGs)
TOKEN=$(az staticwebapp secrets list -n anbennar-worldmap -g rg-anbennar-web --query "properties.apiKey" -o tsv)
swa deploy ./web --deployment-token "$TOKEN" --env production
```

`staticwebapp.config.json` serves `plots.pack` as `application/octet-stream` (so the CDN
does not recompress it and byte-**range** requests work) and sets asset cache headers.
The custom domain is a GoDaddy CNAME `anbennar ‚Üí white-pond-0a0ba4803.7.azurestaticapps.net`
bound with `az staticwebapp hostname set ‚Ä¶ -m cname-delegation` (free managed TLS,
http‚Üíhttps redirect). `plots.pack` is ~15 MB, so the whole site fits the Free tier's
250 MB cap with room to spare.

## Notes

- The map background is the real `terrain.bmp`, cropped to the shipped world extent and
  re-tinted into the dark palette (water / land / hill / peak / snow /
  desert / marsh, classified as in `MapTerrainCodec`). It is projected in
  **web-mercator source-pixel space** ‚Äî the exact inverse of the `lon`/`lat` maps
  `ProvinceExporter` used ‚Äî so the province dots and routes pin to the terrain
  rather than floating over a schematic.
- **Map-label font.** All canvas map labels (province names, geographic tiers, caravan/water
  labels, the Live overlay) use **TeX Gyre Pagella** ‚Äî a free Palatino clone giving the
  Paradox/EU4 map look ‚Äî bundled as `assets/fonts/*.woff2` (regular + bold) and `@font-face`d
  in `styles.css`, so it is pixel-identical across OSes rather than depending on a system
  serif. The single source of the family is `core.LABEL_FONT`. GUST Font License
  (`assets/fonts/GUST-FONT-LICENSE.txt`). `main.mjs` force-loads the faces and redraws, since
  canvas text does not trigger webfont loading on its own.
- This is one directory in the engine's monorepo, not a separate frontend project;
  it has no build tooling beyond Node.
- Interaction & chrome (URL deep links, focus framing, pan clamp, collapsible sidebar,
  floating search, status line, theme) are documented in [`docs/ux.md`](../docs/ux.md);
  the map-rendering pipeline (terrain, shore, sea, icons) in [`docs/coastlines.md`](../docs/coastlines.md).
